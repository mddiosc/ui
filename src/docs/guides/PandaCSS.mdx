# Understanding Panda CSS

Learn how our zero-runtime styling system works.

## ğŸ¼ What is Panda CSS?

Panda CSS is a **zero-runtime CSS-in-JS** library that generates CSS at build time instead of runtime.

### Key Concept

Traditional CSS-in-JS generates CSS in the browser (runtime overhead).

Panda CSS generates CSS at build time (zero runtime overhead).

## ğŸ”„ How It Works

### 1. Configuration

Define design tokens in `panda.config.ts`:

```typescript
export default defineConfig({
  theme: {
    tokens: {
      colors: {
        blue: { 500: { value: '#3b82f6' } }
      }
    }
  }
});
```

### 2. Code Generation

Run `pnpm panda codegen` to generate `styled-system/`:

```bash
pnpm panda codegen
```

### 3. Use in Components

```typescript
import { cva } from '../styled-system/css';

const button = cva({
  base: {
    bg: 'blue.500',
    padding: '4',
  }
});
```

### 4. Build Output

Panda generates optimized CSS:

```css
:root {
  --colors-blue-500: #3b82f6;
}

.bg_blue\.500 {
  background: var(--colors-blue-500);
}
```

## ğŸ¨ Core Functions

### cva() - Component Variants

```typescript
import { cva } from '../styled-system/css';

const button = cva({
  base: {
    padding: '2',
    borderRadius: 'md',
  },
  variants: {
    variant: {
      primary: { bg: 'blue.500' },
      secondary: { bg: 'gray.200' },
    }
  }
});
```

### css() - Basic Styling

```typescript
import { css } from '../styled-system/css';

const className = css({
  padding: '4',
  background: 'blue.500',
});
```

### token() - Access Tokens

```typescript
import { token } from '../styled-system/tokens';

const color = token('colors.blue.500');
```
